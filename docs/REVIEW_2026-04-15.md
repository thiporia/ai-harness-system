# 프로젝트 현황 리뷰 (2026-04-15)

> 지침 기준 대비 현재 구현 상태를 점검한다.

---

## 1. Planner Agent

**지침**: 다른 AI에게 기획을 검토받아야 한다.

| 항목 | 현황 |
|------|------|
| 기획 생성 | ✅ `planner.ts` - LLM 기반 JSON 플랜 생성 |
| 다른 AI의 기획 검토 | ❌ 없음. Reviewer는 테스트 실패 로그에만 반응. Planner 결과에 대한 cross-review 단계 미구현 |

---

## 2. Designer Agent

**지침**: Planner에게 검토받아야 하며, `awesome-design-md`에서 적합한 디자인 `.md` 파일을 확보해서 참고해야 한다.

| 항목 | 현황 |
|------|------|
| 디자인 생성 | ✅ `designer.ts` - plan 기반 컴포넌트 구조 생성 |
| Planner 검토 수신 | ❌ 없음. Designer는 plan을 그대로 입력으로 받을 뿐, Planner가 design을 검토하는 단계 없음 |
| awesome-design-md 참조 | ❌ 없음. `https://github.com/VoltAgent/awesome-design-md` 연동 전혀 없음 |

---

## 3. Developer Agent

**지침**: 현재 가용 가능한 가장 stable한 시스템을 구축한다.

| 항목 | 현황 |
|------|------|
| 코드 생성 | ⚠️ `developer.ts` - LLM 기반 코드 생성 가능 |
| 입력 컨셉 반영 | ❌ 프롬프트가 "React Todo App" 하드코딩. 사용자 입력 컨셉 미반영 |
| 스택 안정성 | ⚠️ Harness context를 주입하나, Capacitor 기반 멀티플랫폼 코드 생성은 실제 보장되지 않음 |

---

## 4. Tester Agent

**지침**: 실제 빌드 후 기동하여 테스트까지 진행. 멀티플랫폼 온전히 대응해야 한다.

| 항목 | 현황 |
|------|------|
| 정적 코드 검사 | ✅ `tester.ts` - 정규식 기반 코드 패턴 검사 |
| 실제 빌드 실행 | ❌ 없음. `build:artifact`는 ts.transpileModule 정적 트랜스파일일 뿐 |
| 앱 기동 및 동작 테스트 | ❌ 없음 |
| 멀티플랫폼 테스트 (Web/Android/iOS) | ❌ 없음. Capacitor 빌드 및 기동 테스트 미구현 |

---

## 5. Reviewer Agent

**지침**: 모든 스텝에서 접근 가능하며 검토를 진행하여 담당 Agent에게 개선을 요청할 수 있다.

| 항목 | 현황 |
|------|------|
| 테스트 실패 시 리뷰 | ✅ `reviewer.ts` - 테스트 실패 로그 분석 후 피드백 |
| Planner 단계 리뷰 | ❌ 없음 |
| Designer 단계 리뷰 | ❌ 없음 |
| Developer 단계 리뷰 | ❌ 없음 (테스트 실패 간접 경유만 존재) |
| 최대 5회 검토 반복 | ✅ `MAX_DEVELOPMENT_RETRIES = 5` |

---

## 6. 공통 / 오케스트레이터

| 항목 | 현황 |
|------|------|
| 사용자 컨셉 입력 | ❌ `DEFAULT_APP_INPUT = "React Todo App with CRUD"` 하드코딩. CLI 인자 미처리 |
| 문서 아티팩트 저장 | ✅ `docs/artifacts/latest-*.md`, `history/*` 저장 |
| 환경변수 기반 LLM 선택 | ✅ OpenAI / Gemini 전환 가능 |

---

## 요약: 지침 충족도

| Agent | 지침 충족 |
|-------|-----------|
| Planner | 🔴 기획 cross-review 없음 |
| Designer | 🔴 Planner 검토 없음 + awesome-design-md 미연동 |
| Developer | 🟡 코드 생성은 되나 컨셉 하드코딩 |
| Tester | 🔴 정적 검사만, 실제 빌드/기동/멀티플랫폼 없음 |
| Reviewer | 🟡 테스트 단계만 작동, 전 단계 접근 불가 |

---

## 우선 개선 제안

1. **[HIGH]** Tester - 실제 `vite build` 또는 `tsc` 실행 + dev server 기동 후 playwright/puppeteer E2E
2. **[HIGH]** Designer - awesome-design-md GitHub 레포 fetch 후 관련 파일 참조
3. **[HIGH]** Planner cross-review - Reviewer를 plan 단계에도 호출
4. **[MEDIUM]** Reviewer 전 단계 통합 - orchestrator에서 각 단계 후 reviewer 호출 옵션
5. **[MEDIUM]** CLI 인자 처리 - `process.argv[2]` 로 사용자 컨셉 입력 수신
